## 💡 使用建议

### 选择框架的三大原则

1. **问题驱动，而非框架驱动**
   - ❌ "我们做个SWOT分析吧"
   - ✅ "我们要解决[问题]，哪个框架最合适？"

2. **组合使用，而非单一框架**
   - 通常需要3-5个框架组合
   - 由外到内：宏观 → 行业 → 内部

3. **框架是工具，洞见是目标**
   - 框架 → 信息收集 → 洞见提炼 → 战略制定 → 行动计划

### 新手快速上手路径

**第1周**：
- Day 1-2: [SWOT分析](./swot-analysis.md)

- 实践：用这3个框架分析一家公司

**第3-4周**：
- 进阶学习6个框架
- 学习框架组合使用

## ❓ 常见问题

### Q1: 我应该全部学会吗？

**A**: 不必。建议：
- **新手**: 先掌握3个（SWOT、五力、商业模式画布）
- **进阶**: 再学6个（价值链、PEST、安索夫、BCG、7S、VRIO）
- **高级**: 根据工作需要选择其他框架

### Q2: 一个项目需要用几个框架？

**A**: 视复杂度而定：
- **快速咨询**: 1-2个（30分钟）
- **中度分析**: 3-4个（2-3小时）
- **深度分析**: 5-7个（1-2天）

### Q3: 框架之间有冲突怎么办？

**A**: 框架不会冲突，它们从不同角度分析：
- PEST: 宏观环境（外部）
- 五力: 行业竞争（外部）
- SWOT: 内外部综合
- 价值链: 内部能力
- 商业模式画布: 整体设计

组合使用，相互补充。

### Q4: 如何判断框架用对了？

**A**: 检查清单：
- [ ] 框架帮助我回答了核心问题
- [ ] 得出了有价值的洞见（而非填空）
- [ ] 洞见指导了具体行动
- [ ] 分析逻辑清晰、有说服力
- [ ] 利益相关方认可分析结果
