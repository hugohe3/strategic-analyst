# 📚 Strategic Analyst 学习路径

> **系统化学习战略分析**：从零基础到精通，7天入门，30天精通

---

## 🎯 学习路径总览

```
学习阶段：

第一阶段：7天入门（每天1-2小时）
├─ 目标：理解核心概念，掌握基础框架
├─ 输出：能独立完成简单的战略分析
└─ 适合：完全新手

第二阶段：30天精通（每天1-2小时）
├─ 目标：掌握所有主要框架，形成分析套路
├─ 输出：能完成复杂的战略分析项目
└─ 适合：有一定基础的学习者

第三阶段：持续实践（持续）
├─ 目标：在实际工作中应用，形成个人风格
├─ 输出：成为战略分析专家
└─ 适合：进阶学习者
```

---

## 📅 7天入门计划

### Day 1：理解结构化思维（1.5小时）

**学习目标**：
- 理解什么是结构化思维
- 掌握MECE原则的基本概念
- 了解金字塔原则

**学习内容**：
1. 阅读 [结构化思维入门指南](./docs/methodology/structured-thinking-guide.md)（45分钟）
   - 重点：三大核心原则
   - 重点：四大工具

2. 阅读 [MECE原则详解](./docs/methodology/mece-principle.md)（30分钟）
   - 重点：MECE的两个要求
   - 重点：四大切分方法

**实践练习**：
```
练习1：日常问题结构化（15分钟）
问题：周末去哪里玩？

要求：
□ 用MECE原则列出所有选项
□ 用矩阵分析评估各选项
□ 做出决策并说明理由

练习2：信息分类（15分钟）
任务：将以下信息分类（MECE）
- 公司有100名员工
- 销售额1000万
- 客户满意度85%
- 市场份额10%
- 研发投入200万
- 员工流失率15%

要求：
□ 按照MECE原则分类
□ 至少用2种不同的分类方式
```

**检验标准**：
- [ ] 能解释什么是结构化思维
- [ ] 能说出MECE的两个要求
- [ ] 能用MECE原则对事物进行分类
- [ ] 完成2个练习

---

### Day 2：掌握第一个框架 - SWOT（1.5小时）

**学习目标**：
- 理解SWOT分析的原理
- 能够独立完成SWOT分析
- 掌握SWOT策略矩阵

**学习内容**：
1. 阅读 [SWOT分析详解](./docs/frameworks/swot-analysis.md)（40分钟）
   - 重点：四个象限的含义
   - 重点：SWOT策略矩阵

2. 观看案例 [外卖平台竞争分析](./examples/mini-cases/competitive-analysis-food-delivery.md)（30分钟）
   - 重点：SWOT如何应用

**实践练习**：
```
练习：分析一家你熟悉的公司（20分钟）

选择：星巴克/苹果/特斯拉/小米（任选一家）

要求：
□ 列出至少3个优势（S）
□ 列出至少3个劣势（W）
□ 列出至少3个机会（O）
□ 列出至少3个威胁（T）
□ 制定至少2个策略（SO、WO、ST、WT各1个）

提示：
- 优势/劣势要具体，不要泛泛而谈
- 机会/威胁要基于外部环境
- 策略要可执行
```

**检验标准**：
- [ ] 能解释SWOT四个象限
- [ ] 能区分内部（SW）和外部（OT）
- [ ] 能制定SWOT策略
- [ ] 完成练习并自评质量

---

### Day 3：掌握行业分析 - 波特五力（2小时）

**学习目标**：
- 理解行业竞争的五种力量
- 能够评估行业吸引力
- 掌握竞争战略制定方法

**学习内容**：
1. 阅读 [波特五力详解](./docs/frameworks/porter-five-forces.md)（50分钟）
   - 重点：五种力量的含义
   - 重点：如何评估行业吸引力

2. 阅读 [框架快速选择指南](./docs/frameworks/QUICK-SELECTION.md)（20分钟）
   - 重点：何时使用波特五力

**实践练习**：
```
练习：分析一个行业（50分钟）

选择：在线教育/外卖/共享单车/新能源车（任选一个）

要求：
□ 分析五种力量（每个力量评分1-5星）
  • 供应商议价能力：[分析] [评分]
  • 买方议价能力：[分析] [评分]
  • 现有竞争者：[分析] [评分]
  • 潜在进入者：[分析] [评分]
  • 替代品威胁：[分析] [评分]

□ 综合评估行业吸引力（高/中/低）
□ 提出竞争战略建议（2-3条）

提示：
- 每个力量要有具体分析，不只是打分
- 要有数据或事实支撑
- 结论要明确
```

**检验标准**：
- [ ] 能说出五种力量是什么
- [ ] 能评估每种力量的强弱
- [ ] 能得出行业吸引力结论
- [ ] 完成练习

---

### Day 4：掌握商业模式设计（2小时）

**学习目标**：
- 理解商业模式画布的九个模块
- 能够描述一个商业模式
- 能够设计新的商业模式

**学习内容**：
1. 阅读 [商业模式画布详解](./docs/frameworks/business-model-canvas.md)（60分钟）
   - 重点：九个模块的含义
   - 重点：模块之间的关系

2. 观看案例 [SaaS市场进入](./examples/mini-cases/market-entry-saas-education.md)（40分钟）
   - 重点：商业模式如何设计

**实践练习**：
```
练习：设计一个商业模式（20分钟）

场景：你想开一家咖啡店

要求：
□ 价值主张：解决什么问题？
□ 客户细分：服务谁？
□ 渠道通路：如何触达客户？
□ 客户关系：如何维护？
□ 收入来源：如何赚钱？
□ 关键资源：需要什么资源？
□ 关键业务：做什么？
□ 关键合作：与谁合作？
□ 成本结构：主要成本是什么？

提示：
- 每个模块要具体
- 要考虑模块之间的一致性
- 可以参考星巴克等成功案例
```

**检验标准**：
- [ ] 能说出九个模块是什么
- [ ] 能描述一个商业模式
- [ ] 能设计简单的商业模式
- [ ] 完成练习

### Day 5：框架组合使用（2小时）

**学习目标**：
- 理解为什么要组合使用框架
- 掌握框架组合的方法
- 能够选择合适的框架组合

**学习内容**：
1. 阅读 [框架组合方案库](./docs/frameworks/framework-combinations.md)（40分钟）
   - 重点：8个常见场景的框架组合
   - 重点：框架选择原则

2. 阅读完整案例 [市场进入策略](./examples/mini-cases/market-entry-saas-education.md)（60分钟）
   - 重点：如何组合使用多个框架
   - 重点：框架之间如何衔接

**实践练习**：
```
练习：设计框架组合方案（20分钟）

场景：你的公司要进入一个新市场

要求：
□ 选择3-5个框架
□ 说明选择理由
□ 设计分析顺序
□ 说明每个框架解决什么问题

参考：
- 市场进入通常用：PEST → 五力 → SWOT → 商业模式画布
- 但要根据具体情况调整
```

**检验标准**：
- [ ] 能说出为什么要组合使用框架
- [ ] 能选择合适的框架组合
- [ ] 能设计分析顺序
- [ ] 完成练习

## 💡 学习建议

### 学习方法

**1. 理论与实践结合**
```
学习循环：
理论学习（40%）→ 案例研究（30%）→ 实践练习（30%）

不要：
❌ 只看理论，不做练习
❌ 只做练习，不看案例
❌ 只看案例，不做项目

要做：
✓ 每学一个框架，立即练习
✓ 每周至少做1个完整案例分析
✓ 每月完成1个真实项目
```

**2. 循序渐进**
```
学习顺序：
简单 → 复杂
单一 → 组合
```

## ❓ 常见问题

### Q1：7天真的能入门吗？

**A**：可以，但需要：
- 每天投入1-2小时（不能间断）
- 认真完成每个练习
- 不要跳过任何步骤
- 7天后能掌握基础，但需要持续练习

### Q2：我工作很忙，能否调整学习节奏？

**A**：可以灵活调整：
- 7天计划可以延长到14天（每天30分钟）
- 30天计划可以延长到60天
- 关键是保持连续性，不要中断太久

### Q3：学完后如何保持？

**A**：持续实践是关键：
- 在工作中应用（每周至少1次）
- 定期复习（每月回顾一次）
- 教别人学习（最好的学习方式）
- 关注行业动态（持续学习）

### Q4：可以跳过某些内容吗？

**A**：不建议跳过基础内容：
- Day 1-3是基础，必须学习
- Day 4-6可以根据需要调整
- Day 7的总结很重要，不要跳过

### Q5：学习遇到困难怎么办？

**A**：多种途径获取帮助：
- 重读相关文档
- 查看更多案例
- 使用AI Agent辅助理解
- 在GitHub提Issue
- 与他人讨论交流
