## 🎯 我们需要什么？

### ⭐⭐⭐⭐⭐ P0：最需要的贡献

**1. Mini-Case（完整案例）**
- 不同行业/场景的战略分析案例
- 必须展示完整的五阶段流程
- 真实场景（已脱敏处理）
- 字数：5000-15000字

**2. 方法论文档**
- 补充缺失的框架（PEST、价值链、安索夫矩阵等）
- 优化现有文档（增加案例、图表、实践指南）
- 字数：3000-8000字

### ⭐⭐⭐ P1：很有价值

**3. 使用反馈**
- 使用 AI Agent 的体验反馈
- 发现的 Bug 和改进建议
- AI 输出质量评价

**4. 文档改进**
- 错别字、语法修正
- 表述优化
- 增加示例和说明

### ⭐ P2：欢迎但非紧急

**5. 其他**
- 新功能建议（通过 Issue 提出）
- 设计优化建议

**❌ 不再接受**：
- Python/Excel 工具代码（项目已移除 tools/ 目录）
- 不完整的案例片段

## 🚀 5分钟贡献一个 Mini-Case

### Step 1：准备案例内容

**案例要求**：

| 项目 | 要求 |
|------|------|
| **场景** | 真实或基于真实场景（已脱敏） |
| **结构** | 必须包含五个阶段 |
| **质量** | 逻辑清晰，有实际价值 |
| **字数** | 5000-15000字 |
| **格式** | Markdown |

**参考模板**：复制任一现有案例，替换内容
- [市场进入策略](./examples/mini-cases/market-entry-saas-education.md)
- [个人职业规划](./examples/mini-cases/career-planning-developer.md)
- [竞争分析](./examples/mini-cases/competitive-analysis-food-delivery.md)

---

### Step 2：案例内容结构

```markdown
# 案例：[标题]

## 案例背景
- 公司/个人基本情况
- 当前问题/困惑

## 五阶段分析过程

### 阶段 1：问题诊断
- AI Agent 与用户的对话
- 6要素问诊
- 5WHY分析
- 问题定义确认书

### 阶段 2：结构化拆解
- MECE分析框架
- 多维度拆解
- 关键未知清单

### 阶段 3：假设形成
- 2-3个战略假设
- 逻辑链
- 决策矩阵

### 阶段 4：验证迭代
- 验证计划
- 成功标准

### 阶段 5：方案交付
- 执行摘要
- 详细方案
- 财务预测
- 风险应对
- 行动计划

## 案例总结
- 关键学习点
- AI Agent 的价值
- 适用场景
```

### Step 3：提交流程

**1. Fork 仓库**
```bash
# 在 GitHub 上点击 Fork 按钮
```

**2. 创建分支**
```bash
git clone https://github.com/your-username/strategic-analyst.git
cd strategic-analyst
git checkout -b case/your-case-name
```

**3. 添加案例文件**
```bash
# 创建案例文件
vi examples/mini-cases/your-case-name.md
# 按照模板编写内容
```

**4. 更新索引**
```markdown
# 编辑 examples/README.md，添加：

6. **[你的案例标题](./mini-cases/your-case-name.md)** ⭐⭐⭐⭐
   - 场景：简短描述
   - 亮点：核心价值
   - 时长：XX分钟阅读
```

**5. 提交 Pull Request**
```bash
git add examples/mini-cases/your-case-name.md examples/README.md
git commit -m "新增案例: your-case-name"
git push origin case/your-case-name
```

然后在 GitHub 上创建 Pull Request。

## ✅ PR 审查标准

### Mini-Case 审查清单

- [ ] **结构完整**：包含五个阶段
- [ ] **质量**：逻辑清晰，有实际价值
- [ ] **原创**：不是简单复制他人案例
- [ ] **脱敏**：无真实公司名、人名、敏感数据
- [ ] **格式**：Markdown 格式正确，无明显错误
- [ ] **索引更新**：已在 `examples/README.md` 中添加

### 方法论文档审查清单

- [ ] **内容完整**：包含定义、方法、案例、局限性
- [ ] **至少2个案例**：真实、详细、有说服力
- [ ] **图表**：关键概念有配图或表格
- [ ] **引用**：理论部分有参考来源
- [ ] **格式规范**：Markdown 格式正确

## ❓ 常见问题

### Q1：我的案例可以是虚构的吗？

**A**：可以，但必须基于真实场景逻辑，不能脱离实际。最好是真实案例脱敏处理。

### Q2：案例必须是成功的吗？

**A**：不一定。失败案例也有价值，重点是分析过程和学习点。

### Q3：我不是咨询顾问，可以贡献吗？

**A**：可以！只要案例逻辑清晰、有实际价值即可。个人决策案例（如职业规划）也很有价值。

### Q4：我的案例涉及敏感信息怎么办？

**A**：必须脱敏处理：
- 公司名用化名（如"智学云"）
- 人名用化名（如"李明"）
- 敏感数据模糊化或替换
- 保留问题本质和分析逻辑

### Q5：案例被拒绝了怎么办？

**A**：不要灰心！我们会说明原因和改进建议。修改后可以重新提交。

### Q6：贡献方法论文档需要学术水平吗？

**A**：不需要。我们需要的是**实用指南**，不是学术论文。重点是：
- ✅ 清晰易懂
- ✅ 有真实案例
- ✅ 能指导实践
