## 🎯 四大基本原则（贯穿所有阶段）

### 原则1：数字说话 📊 (Fact Based)

**核心要求**：
- 假设所有未经检验的数字都可能是错误的
- 具备常识推理能力，快速估算验证数据合理性
- 关注少数特例和反常数据（可能是关键洞见的来源）
- 承认数字和逻辑的局限性（黑天鹅事件）

**实践方法**：
- 每个结论必须有数据支撑
- 标注数据来源和可信度
- 区分"精确数字"和"粗略估算"
- 对异常数据保持好奇和警惕

**示例**：
```
❌ 错误："市场很大"
✅ 正确："根据艾瑞咨询2024年报告，目标市场规模为120亿元，
         年增长率18%（2020-2024数据）"
```

---

### 原则2：洞见优于表象 💡 (Insight Driven)

**核心要求**：
- 洞见是连接所有表象的根本原因
- 洞见具有行为导向性（能指导行动）
- 沟通中先说结论和洞见，再陈述论据
- 深挖"下一层面的细节"，避免循环论证

**判断标准**：
```
❌ 表象："销售额下降了20%"
✅ 洞见："销售额下降的根本原因是核心客户流失率上升，
         而流失率上升源于竞品推出了更有竞争力的价格方案，
         建议立即调整定价策略并加强客户关系管理"
```

**三个层次**：
1. **数据层**：发生了什么（What）
2. **原因层**：为什么发生（Why）
3. **洞见层**：意味着什么，应该怎么做（So What）

---

### 原则3：MECE原则 🧩 (Mutually Exclusive, Collectively Exhaustive)

**核心要求**：
- 确保分析维度相互独立（Mutually Exclusive）
- 确保分析维度完全穷尽（Collectively Exhaustive）
- 批判性审视经典理论框架（不迷信权威）
- MECE是催生突破性思维的工具，而非限制

**MECE检验清单**：
```
□ 各维度之间有重叠吗？（相互独立）
□ 是否遗漏了重要维度？（完全穷尽）
□ 维度划分的逻辑是什么？（清晰原则）
□ 是否存在更好的切分方式？（优化思维）
```

**常见错误**：
```
❌ 不符合MECE：
   客户分类 = 大客户 + 小客户 + 重要客户
   （重叠：大客户可能也是重要客户）

✅ 符合MECE：
   方式1（按规模）：大客户 + 中客户 + 小客户
   方式2（按价值）：战略客户 + 重点客户 + 普通客户
```

---

### 原则4：假设为前提 🧐 (Hypothesis Driven)

**核心要求**：
- 先提出有依据的假设，再收集数据验证
- 大胆假设，仔细求证
- 区分"认为"（主观猜测）和"知道"（客观事实）
- 避免"先入为主"，保持开放心态接受假设被推翻

**实践方法**：
- 形成假设时标注"认为"
- 验证后的结论标注"知道"
- 清晰说明假设的前提条件
- 主动寻找可能推翻假设的证据（避免确认偏误）

**假设陈述模板**：
```
假设：[一句话假设]

逻辑链：
  前提（知道）→ 推理1（认为）→ 推理2（认为）→ 结论（认为）

支撑依据（已知）：
  • [事实1]
  • [数据2]

关键假设（需验证）：
  • [假设1]
  • [假设2]

前提条件：
  如果 [条件1] 和 [条件2] 成立，则假设成立

验证方式：
  通过 [方法] 收集 [数据]，如果发现 [证据]，则假设成立
```

## 🔧 MECE切分四大招式

### 招式1：公式法

**原理**：利用公认的数学公式或商业公式进行分解

**示例**：
```
利润问题 = 收入 - 成本
         ↓
收入 = 价格 × 销量
成本 = 固定成本 + 可变成本

进一步分解：
销量 = 新客户数量 + 老客户复购
价格 = 基础价格 + 溢价
```

**适用场景**：
- 财务分析
- 运营效率分析
- 增长问题分析
- 任何有公式的问题

---

### 招式2：子目录列举法

**原理**：穷尽列举下一层所有可能的子类别

**示例**：
```
市场分析 = 客户 + 竞争 + 渠道 + 产品
         ↓
客户 = 企业客户 + 个人客户 + 政府客户
     ↓
企业客户 = 大型企业 + 中型企业 + 小型企业
```

**适用场景**：
- 市场分析
- 用户画像
- 产品线规划
- 组织结构设计

**注意事项**：
- 确保分类标准统一
- 避免遗漏重要类别
- 控制层级深度（不超过3-4层）

---

### 招式3：流程法

**原理**：按照事物发展的时间顺序或流程切分

**示例**：
```
客户生命周期 = 获客 → 激活 → 留存 → 变现 → 推荐
              (AARRR模型)

产品开发流程 = 需求 → 设计 → 开发 → 测试 → 上线 → 迭代

销售流程 = 线索获取 → 资格审查 → 需求分析 → 方案呈现 
         → 谈判 → 成交 → 售后
```

**适用场景**：
- 流程优化
- 用户旅程分析
- 项目管理
- 时间序列问题

---

### 招式4：逻辑框架法

**原理**：使用逻辑对立面或经典框架切分

**示例**：
```
影响因素 = 内部因素 + 外部因素
         ↓
内部 = 优势 + 劣势
外部 = 机会 + 威胁
(SWOT框架)

其他常见对立：
- 定性 vs 定量
- 短期 vs 长期
- 存量 vs 增量
- 可控 vs 不可控
```

**适用场景**：
- 战略规划
- 环境分析
- 风险评估
- 综合分析

## 📐 3-3原则（多维深度分解）

**要求**：用3个不同维度，进行至少3层深度分解

```
问题
├── 维度1：[第一个切分角度]
│   ├── 子维度1.1
│   │   ├── 细节1.1.1
│   │   └── 细节1.1.2
│   └── 子维度1.2
│       ├── 细节1.2.1
│       └── 细节1.2.2
│
├── 维度2：[第二个切分角度]
│   ├── 子维度2.1
│   └── 子维度2.2
│
└── 维度3：[第三个切分角度]
    ├── 子维度3.1
    └── 子维度3.2
```

**为什么是3？**
- 人脑短期记忆容量：7±2
- 3个维度既全面又可管理
- 3层深度能够触达本质

**实践建议**：
- 第1层：宏观框架（不超过5个维度）
- 第2层：关键要素（每个维度3-5个子维度）
- 第3层：具体细节（根据需要展开）

## 🗣️ 金字塔原则（结构化表达）

### 核心结构

```
                  核心结论
                     ↓
        ┌────────────┼────────────┐
      论据1        论据2         论据3
        ↓            ↓             ↓
    支撑点1.1    支撑点2.1     支撑点3.1
    支撑点1.2    支撑点2.2     支撑点3.2
```

### 三大原则

**1. 结论先行**
```
❌ 错误："我们调研了100个客户，分析了竞品价格，研究了行业报告，
         最后发现应该降价10%"

✅ 正确："建议降价10%
         
         理由：
         1. 竞品价格低15%（100客户调研，80%认为价格偏高）
         2. 价格敏感度测试显示10%是临界点
         3. 降价后仍可保持20%毛利率"
```

**2. 归类分组**
- 同一组的内容属于同一逻辑范畴
- 每组3-5个要点
- 使用并列关系（而非包含关系）

**3. 逻辑递进**
- 时间顺序：第一、第二、第三
- 结构顺序：从整体到部分
- 重要性顺序：从高到低

## 🎯 5WHY追问法（深挖根因）

**使用场景**：当发现问题可能是表象时

**操作方法**：
```
表象问题：销售额下降

Why 1：为什么销售额下降？
→ 因为客户流失率上升

Why 2：为什么客户流失率上升？
→ 因为客户满意度下降

Why 3：为什么客户满意度下降？
→ 因为产品质量出现问题

Why 4：为什么产品质量出现问题？
→ 因为供应商更换后原材料质量下降

Why 5：为什么要更换供应商？
→ 因为采购部门单纯追求成本降低（根本原因）

真实问题：采购决策机制问题，而非销售问题
```

**停止条件**：
- 找到可以直接采取行动的根本原因
- 继续追问会进入另一个问题域
- 已经触达组织/系统层面的问题

## 🧠 十个核心工作习惯

### 1. 反对的责任
提出不同意见是责任，沉默即失职

### 2. 解决正确的问题
时刻反思问题定义是否正确

### 3. 下一层面的细节
拒绝循环论证，习惯性深挖一层

### 4. 总结提炼（神奇数字"3"）
将海量信息合成为简洁洞见，每次输出不超过3个核心要点

### 5. 第一天的答案
快速形成初步假设和框架，不拖延

### 6. 问正确的问题
提出挑战固有成见的"美丽问题"

### 7. 认为还是知道
清晰区分主观猜测和客观事实

### 8. 移动时间轴
思考"然后呢？"，预演决策的未来后果

### 9. 数字和逻辑
作为思考和沟通的基石

### 10. 知道边界
认识自我认知和知识的局限性，保持谦逊
