## 🤖 方式1：使用AI角色辅助（推荐新手）

### Step 1：选择AI工具

支持的AI工具：
- **ChatGPT**（推荐GPT-4）
- **Claude**（推荐Claude 3.5 Sonnet）
- **Cursor**（推荐用于编程相关咨询）
- 其他支持长上下文的AI助手

### Step 2：加载角色提示词

1. 打开文件：`roles/strategic_analyst.md`

2. 复制全部内容

3. 在AI工具中**新建对话**，粘贴内容

4. AI将回复准备就绪的确认信息

### Step 3：开始咨询

**示例对话**：

```
您：我需要为公司制定明年的市场拓展战略

AI：您好！我是结构化战略分析专家。👋

【任务接收】阶段0

📥 我注意到您需要：制定市场拓展战略

🔍 初步判断：
  • 任务类型：战略规划类
  • 复杂度：中高
  • 建议流程：阶段1 → 阶段2 → 阶段3 → 阶段5

💡 我建议采用以下工作方式：
  1. 先花5-10分钟明确问题（阶段1：问题诊断）
  2. 再用15-20分钟建立分析框架（阶段2：结构化拆解）
  3. 然后提出初步方案（阶段3：假设形成）
  4. 最后形成完整建议（阶段5：方案交付）

⏱️ 预计耗时：1-2小时（分多轮对话完成）

您认可这个工作方式吗？让我们从【阶段1：问题诊断】开始？
```

### Step 4：跟随AI的引导

AI将：
- ✅ 采用苏格拉底式提问引导您思考
- ✅ 分阶段推进，每个阶段都会暂停确认
- ✅ 提供结构化的输出（确认书、框架图、方案等）
- ✅ 在关键节点要求您确认后再继续

**💡 提示**：
- 回答AI的问题要具体，避免模糊表述
- 如果AI的理解有误，及时纠正
- 可以随时要求AI返回上一阶段或深入某个点

## 📝 方式2：直接使用模板

### Step 1：选择合适的模板

进入 `templates/stage-templates/` 目录：

| 模板 | 使用场景 | 填写耗时 |
|------|---------|---------|
| `stage1-problem-definition.md` | 需要明确问题 | 30-60分钟 |
| `stage2-analysis-framework.md` | 需要建立分析框架 | 1-2小时 |
| `stage3-hypothesis-generation.md` | 需要提出解决方案 | 2-3小时 |
| `stage4-validation-plan.md` | 需要验证假设 | 1-2小时 |
| `stage5-strategic-report.md` | 需要形成完整报告 | 3-5小时 |

### Step 2：复制模板到您的项目

```bash
# 创建项目目录
mkdir my-project
cd my-project

# 复制需要的模板
cp ../consulthub/templates/stage-templates/stage1-problem-definition.md ./

# 或者复制全部模板
cp -r ../consulthub/templates/stage-templates/* ./
```

### Step 3：填写模板

1. 打开模板文件
2. 查找所有 `[...]` 标记（需要填写的内容）
3. 根据提示填写
4. 删除不适用的部分
5. 保留必要的结构

**💡 提示**：
- 每个模板顶部都有"使用说明"和"填写要点"
- 模板中的示例（灰色部分）可以参考
- 填写时遵循MECE原则和金字塔原则

### Step 4：客户确认

- 每个阶段完成后，必须得到客户/决策者确认
- 使用模板底部的"客户确认"部分
- 确认后再进入下一阶段

```
第1步：理解核心概念（30分钟）
  ↓
第2步：学习五步法（1小时）
  ↓
第3步：掌握分析工具（1小时）
  ↓
第4步：实战演练（1-2小时）
```

### 第1步：理解核心概念（30分钟）

**必读文档**：
1. `roles/strategic_analyst.md` - 完整的角色定义和方法论
2. `docs/methodology/mece-principle.md` - MECE原则详解

## ❓ 常见问题

### Q1：我是新手，应该从哪里开始？

**A**：推荐使用**方式1：AI角色辅助**
1. 打开 `roles/strategic_analyst.md`
2. 复制全部内容到ChatGPT或Claude
3. 直接描述您的问题，AI会引导您

### Q2：模板太长了，可以简化吗？

**A**：可以！保留核心结构，删除"可选"部分：
- 阶段1：保留6要素
- 阶段2：保留MECE框架
- 阶段3：保留逻辑链和风险
- 阶段5：保留执行摘要和行动计划

### Q3：可以跳过某个阶段吗？

**A**：视情况而定：
- ❌ 不建议跳过：阶段1（问题定义是基础）
- ✅ 可以跳过：阶段4（如果时间紧迫）
- ⚠️ 谨慎跳过：阶段2、3、5

### Q4：如何判断分析是否符合MECE原则？

**A**：使用MECE检验清单（详见 `docs/methodology/mece-principle.md`）：
- [ ] 各部分之间有重叠吗？（ME）
- [ ] 分类标准统一吗？
- [ ] 有遗漏吗？（CE）
- [ ] 覆盖了所有重要方面吗？

### Q5：团队如何协作使用这套工具？

**A**：建议工作方式：
1. **项目负责人**：使用AI角色或模板主导分析
2. **团队成员**：分工负责不同维度的深入分析
3. **定期同步**：使用阶段性输出物进行对齐
4. **版本控制**：使用Git管理模板和文档

### Q6：这套方法适用于哪些场景？

**A**：广泛适用于：
- ✅ 企业战略规划
- ✅ 市场拓展决策
- ✅ 产品策略制定
- ✅ 组织架构优化
- ✅ 政府项目谋划
- ✅ 个人职业规划

**不太适用于**：
- ❌ 纯技术实现（需要技术专家）
- ❌ 紧急救火（需要快速决策，来不及系统分析）
- ❌ 琐碎事务（使用结构化思维的成本过高）

## 💡 实用技巧

### 技巧1：从小项目开始练习

**建议**：
- 不要一上来就用在重大战略决策上
- 先用在小型项目上练手（如部门OKR制定）
- 熟练后再应用到复杂问题

### 技巧2：善用"神奇数字3"

**原则**：
- 每个层级不超过5个要点（理想是3个）
- 3个维度、3层深度（3-3原则）
- 3个核心洞见、3-5个战略建议

**为什么**：
- 人脑短期记忆容量有限（7±2）
- 3个要点最易于记忆和沟通
- 过多要点会稀释重点

### 技巧3：结论先行，证据后置

**错误示范**：
```
我们调研了100个客户，分析了竞品价格，研究了行业报告，
最后发现应该降价10%。
```

**正确示范**：
```
建议降价10%。

理由：
1. 竞品价格低15%（调研100客户，80%认为价格偏高）
2. 价格敏感度测试显示10%是临界点
3. 降价后仍可保持20%毛利率
```

### 技巧4：多用可视化

**推荐工具**：
- 表格：对比数据、罗列清单
- 矩阵：多维度分析（如BCG矩阵）
- 流程图：展示流程或逻辑链
- 树状图：展示MECE分解

**原则**：
- 一图胜千言
- 降低阅读成本
- 提高记忆度
