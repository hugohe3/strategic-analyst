# 🤖 Agent 系统指南

> Strategic Analyst - AI 驱动的战略分析专家系统

## 📋 概述

本项目提供了一个**结构化战略分析专家 Agent**，基于麦肯锡式咨询方法论，能够通过五阶段交互式工作法协助用户完成从问题诊断到方案交付的完整战略分析流程。

### Agent 核心能力

- 🎯 **结构化问题诊断** - 运用 6 要素法和 5WHY 深度挖掘真实问题
- 📊 **MECE 分析框架** - 系统化拆解复杂问题，确保不遗漏关键因素
- 💡 **假设驱动推理** - 快速形成战略假设，设计验证路径
- 🔄 **分阶段迭代** - 用户主导，每阶段确认后推进
- 📈 **数据驱动决策** - 基于事实和逻辑，提供可执行建议

### 阶段转换原则

- ✅ **用户主导** - 只有用户确认后才进入下一阶段
- ✅ **可回溯** - 任何时候都可以返回前面阶段修正
- ✅ **灵活跳跃** - 根据实际情况可以跳过某些阶段
- ✅ **强制停顿** - Agent 不会自行推进到下一阶段

## 💬 Agent 交互指南

### 有效对话技巧

**1. 提供充分上下文**

❌ 模糊提问：

```
"帮我分析一下市场"
```

✅ 清晰提问：

```
"我们是一家B2B SaaS公司，当前月收入500万，
想在未来6个月内拓展到制造业垂直市场，
请帮我分析这个市场进入策略"
```

**2. 及时反馈和确认**

Agent 每个阶段都会暂停等待确认：

```
Agent：...以上是我对问题的理解，请确认：
       1. 是否准确？
       2. 是否有遗漏？
     
✅ 好的回应："准确，我们可以进入下一阶段"
✅ 好的回应："第3点需要修正，我们的时间限制是3个月而非6个月"
❌ 差的回应："继续吧"（没有实质反馈）
```

**3. 主动提供数据和信息**

Agent 会要求提供特定信息，尽可能详细：

```
Agent：请问当前的市场份额是多少？

✅ 好的回应："我们在华东市场份额约15%，华北10%，华南5%"
❌ 差的回应："不太清楚"（尝试提供估算或替代信息）
```

**4. 挑战和质疑 Agent**

鼓励"反对的责任"：

```
Agent：我建议采用价格竞争策略...

✅ 好的回应："我不认同，我们的品牌定位是高端，
             降价会损害品牌形象。有其他方案吗？"
           
Agent 会积极调整方向
```

## 🎯 使用场景示例

### 场景 1：企业战略决策

**问题类型**：市场进入策略、产品战略规划、竞争分析

**Agent 工作流程**：

```
阶段1 → 明确进入目标、成功标准、资源约束
阶段2 → 建立市场分析框架（市场规模、竞争、能力等）
阶段3 → 提出2-3个进入策略（直销/渠道、高端/中端等）
阶段4 → 设计市场调研和试点方案
阶段5 → 交付完整进入策略和实施路线图
```

**预计时长**：15-30 轮对话

**问题类型**：职业转型、教育投资、重大生活选择

```
阶段1 → 明确职业目标、当前状态、限制条件
阶段2 → 建立分析框架（能力、兴趣、市场、资源）
阶段3 → 提出2-3个职业路径选择
```

**Agent 工作流程**：

```
Agent 快速评估 → 跳到阶段3或阶段5 → 输出简化方案
```

**预计时长**：3-5 轮对话

## ⚙️ Agent 配置

### 行为模式

Agent 遵循以下核心原则（内置于角色定义）：

**四大基本原则**：

1. 📊 **数字说话**（Fact based）- 所有结论必须有数据支撑
2. 💡 **洞见优于表象**（Insight driven）- 深挖根本原因
3. 🧩 **MECE 原则**（相互独立，完全穷尽）- 结构化分析
4. 🧐 **假设为前提**（Hypothesis driven）- 先假设后验证

**沟通风格**：

- ✅ 简洁输出（每次 300-600 字，阶段5除外）
- ✅ 结构化呈现（表格、矩阵、逻辑树）
- ✅ 苏格拉底式提问（引导思考而非直接给答案）
- ✅ 批判性思维（鼓励用户挑战分析）

**输出控制**：

- ✅ 结论先行（金字塔原则）
- ✅ 分层展示（先框架后细节）
- ✅ 强制停顿（关键节点等待确认）

## 📊 效果评估

### Agent 输出质量标准

每个阶段的输出应达到以下标准：

**阶段1 - 问题定义确认书**：

- ✅ 6要素完整（背景、成功标准、边界、限制、相关方、资源）
- ✅ 问题陈述清晰（一句话说明核心问题）
- ✅ 用户确认"这就是我们要解决的问题"

**阶段2 - 分析框架确认书**：

- ✅ 至少3个维度，每个维度3层拆解（3-3原则）
- ✅ 符合MECE原则（相互独立、完全穷尽）
- ✅ 标注已知、认为、需验证的信息

**阶段3 - 战略假设方案**：

- ✅ 提供2-3个可选方案
- ✅ 每个方案包含：逻辑链、支撑依据、前提条件、预期效果、风险
- ✅ 提供决策矩阵帮助选择

**阶段4 - 验证计划书**：

- ✅ 明确验证目标和关键问题
- ✅ 数据需求清单和获取方法
- ✅ 访谈提纲和调研计划
- ✅ 成功标准（如何判断假设成立/证伪）

**阶段5 - 战略建议报告**：

- ✅ 执行摘要（1页，核心结论+建议+快速行动）
- ✅ 完整方案（问题背景、分析、策略、实施计划）
- ✅ 责任分工（RACI矩阵）
- ✅ 资源预算和ROI评估
- ✅ 风险评估和应对

## 🐛 常见问题

### Q1：Agent 输出太长/太短怎么办？

**调整方法**：

```
您：请简化输出，只给关键要点

或

您：请展开详细说明，特别是[某个部分]
```

### Q2：Agent 理解错了我的问题

**纠正方法**：

```
您：我需要澄清，我的真实问题是...
    [重新陈述]

Agent 会立即调整，不会坚持错误理解
```

### Q3：想跳过某个阶段

**跳阶段方法**：

```
您：我已经有明确的问题定义和分析框架，
    请直接进入阶段3，给我方案建议

Agent 会评估可行性并跳转
```

### Q4：想回到前一阶段修正

**回溯方法**：

```
您：我发现阶段1的问题定义不准确，
    我们需要回到阶段1重新定义

Agent 会立即回溯，不会抗拒
```

### Q5：Agent 没有我需要的行业知识

**应对方法**：

```
您：[提供行业背景资料或数据]
    请基于这些信息进行分析

Agent 会整合您提供的信息进行分析

或者：

您：这个问题涉及[专业领域]，可能超出你的能力范围

Agent 会坦诚说明局限性，提供框架指导而非具体答案
```

## 🎓 最佳实践

### 1. 充分准备

在启动 Agent 前准备好：

- ✅ 背景资料（公司/个人情况）
- ✅ 可用数据（财务、运营、市场数据）
- ✅ 时间预期（多久需要完成）
- ✅ 资源限制（预算、人力等）

### 2. 分段进行

不要试图一次对话完成所有阶段：

- ✅ 阶段1-2可能需要1-2次对话（1-2小时）
- ✅ 阶段3-4可能需要1次对话+线下验证
- ✅ 阶段5需要1次对话整合输出

### 3. 保存中间结果

每个阶段的输出都要保存：

```
# 建议的文件结构
my-project/
├── stage1-problem-definition.md
├── stage2-analysis-framework.md
├── stage3-hypothesis.md
├── stage4-validation-plan.md
└── stage5-final-report.md
```

### 4. 结合模板使用

Agent 输出 → 模板格式化 → 专业交付物：

```
1. Agent 完成阶段5输出
2. 复制到 templates/stage-templates/stage5-strategic-report.md
3. 调整格式，补充细节
4. 生成 PPT/Word 最终交付物
```

### 5. 持续学习

每次使用后复盘：

- ✅ 哪些问题 Agent 处理得好？
- ✅ 哪些地方需要更多人工判断？
- ✅ 学到了什么新的分析方法？
- ✅ 如何优化下次的使用？
